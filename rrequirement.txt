WORKSHOP EVENT MANAGEMENT SYSTEM

Overview:

The Workshop Event Management System is a digital platform designed to streamline the
organization and participation process for various workshops and events. Administrators can
create and manage workshop events by providing essential details such as the event name,
description, field of workshop, duration (in days), organizer name, category.

Users can browse through available workshops and register by submitting their details, including
reasons for attending, professional background, proof image, age, city, occupation and an
optional additional note. Each registration records the user ID, associated event ID, registration
date, and current status (e.g., Submitted, UnderReview, Approved, or Rejected).

The system's intuitive interface enables users to track their registration status, receive event-
related updates, and gain access to enriching workshop experiences.

Application assumptions:

1. The login page should be the first page rendered when the application loads.

2. Manual routing should be restricted by implementing Auth Guard, utilizing the canActivate
interface. For example, if the user enters as http://localhost:8080/dashboard or

http://localhost:8080/user the page should not navigate to the corresponding page instead it
should redirect to the login page.

3. Unless logged into the system, the user cannot navigate to any other pages.

4. Logging out must again redirect to the login page.

Exceptions: (Exceptions / WorkshopEventException.cs)

1. Inside Exceptions folder create the exception file named WorkshopEventException
(WorkshopEventException.cs).
2. Purpose: The WorkshopEventException class provides a mechanism for handling
exceptions related to workshop events operations within the application.

3. Namespace: It should located within the dotnetapp.Exceptions namespace.

4. Inheritance: Inherits from the base Exception class, enabling it to leverage existing
exception handling mechanisms.
5. Constructor: Contains a constructor that accepts a message parameter, allowing to specify
custom error messages when throwing exceptions.

For example, you might throw a WorkshopEventException.

1. When attempting to delete a workshop event that is referenced by a user booking.
2. When trying to add a workshop event with the same event name as an existing one.

Important note:

Implement database logic only in the service file functions without using try-catch. Use try-
catch only in the controller files and call the service file functions inside it.

WorkshopEventService (Services / WorkshopEventService.cs)
This service class provides methods to interact with workshop event data stored in the database.

Constructor:
public WorkshopEventService(ApplicationDbContext context)

_context = context;

Functions:

1. public async Task<IEnumerable<WorkshopEvent>> GetAllWorkshopEvents():

a. Retrieves and returns all workshop events from the database.

2. public async Task<WorkshopEvent> GetWorkshopEventById(int workshopEventId):

a. Retrieves a workshop event from the database with the specified workshopEventId.

3. public async Task<bool> AddWorkshopEvent(WorkshopEvent workshopEvent):

a. Checks if a workshop event with the same event name already exists in the database.
b. If a workshop event with the same name exists, throws a WorkshopEventException with
the message "Event with the same name already exists".
c. If no event with the same name exists, adds the new workshop event to the database.
d. Saves changes asynchronously to the database.
e. Returns true for a successful insertion.

4. public async Task<bool> UpdateWorkshopEvent(int workshopEventId,
WorkshopEvent workshopEvent):

a. If no workshop event with the specified workshopEventId is found in the database, returns
false.
b. Checks if a workshop event with the same event name already exists in the database.
c. If a workshop event with the same name exists, throws a WorkshopEventException with
the message "Event with the same name already exists".
d. If no workshop event with the same name exists, updates the existing workshop event with
the values from the provided workshopEvent object.
e. Saves changes asynchronously to the database.
f. Returns true for a successful update.

5. public async Task<bool> DeleteWorkshopEvent(int workshopEventId):

a. Retrieves the workshop event from the database based on the provided workshopEventId.

b. Checks if the workshop event is referenced in any Booking. If referenced, throws a
WorkshopEventException with the message "Workshop event cannot be deleted, it is
referenced in bookings".

c. If the workshop event is not referenced by any Booking, removes the workshop event
from the database.

d. Saves changes asynchronously to the database.

e. Returns true for a successful deletion.

1. AuthService(auth.service.ts):

. Create a service name as auth inside app/services folder to implement the following
functions.

Methods Overview:

. register(user: User): Observable<any>:
. Use this method to register a new user. It sends a POST request to the '/api/register'
endpoint with the user data provided as the body.

. login(login : Login): Observable<any>:
. This method is used to authenticate a user by logging them in. It sends a POST request to
the '/api/login' endpoint with the user's email and password. Upon successful login, it

stores the JWT token in localStorage and updates the user's role and ID using
BehaviorSubjects.

2. WorkshopEventService (workshop-event.service.ts)

Create a service named workshop-event inside app/services and implement the following
functions in it.

Methods Overview:

1. getAllWorkshopEvents(): Observable<WorkshopEvent[]>

a. Fetch all workshop events from the server.

b. Sends a GET request to the /api/workshop-events endpoint with the authorization
token prefixed with 'Bearer' stored in localStorage.

2. addWorkshopEvent(requestObject: WorkshopEvent):
Observable<WorkshopEvent>

a. Adds a new workshop event.

b. Sends a POST request to the /api/workshop-event endpoint with the event data
provided as requestObject and the authorization token prefixed with 'Bearer'
stored in localStorage.

3. deleteWorkshopEvent(workshopEventId: number): Observable<void>

a. Deletes a workshop event with the specified ID.

b. Sends a DELETE request to the /api/workshop-event/{workshopEventId}
endpoint with the authorization token prefixed with 'Bearer' stored in localStorage.

4. getWorkshopEventById(id: number): Observable<WorkshopEvent>

a. Retrieves a workshop event by its ID.

b. Sends a GET request to the /api/workshop-event/{id} endpoint with the
authorization token prefixed with 'Bearer' stored in localStorage.

5. updateWorkshopEvent(id: number, requestObject: WorkshopEvent):
Observable<WorkshopEvent>

a. Updates an existing workshop event.
b. Sends a PUT request to the /api/workshop-event/{id} endpoint with the event ID
and the updated event data provided as requestObject, along with the
authorization token prefixed with 'Bearer' stored in localStorage.

6. getRegisteredEvents(userId: number): Observable<WorkshopEvent|]>

a. Retrieves all workshop events a specific user has registered for.

b. Sends a GET request to the /api/bookings/user/{userId} endpoint with the
authorization token prefixed with 'Bearer' stored in localStorage.

7. addWorkshopEventBooking(data: Booking): Observable<Booking>

a. Submits a new booking for a workshop event.

b. Sends a POST request to the /api/booking endpoint with the booking data
provided as data and the authorization token prefixed with 'Bearer' stored in
localStorage.

8. deleteWorkshopEventBooking(bookingId: number): Observable<void>

a. Deletes a workshop event booking with the specified ID.

b. Sends a DELETE request to the /api/booking/{bookingId} endpoint with the
authorization token prefixed with 'Bearer' stored in localStorage.

9. getAllWorkshopEventBookings(): Observable<Booking|]>

a. Fetches all workshop event bookings from the server.

b. Sends a GET request to the /api/bookings endpoint with the authorization token
prefixed with 'Bearer' stored in localStorage.

10. updateWorkshopEventBooking(id: number, booking: Booking):
Observable<Booking>

a. Updates the details of an existing workshop event booking.

b. Sends a PUT request to the /api booking/{id} endpoint with the booking ID,
updated booking data, and the authorization token prefixed with 'Bearer' stored in
localStorage.
